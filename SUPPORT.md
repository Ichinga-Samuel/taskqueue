# Support

Use the channel that best matches the request.

## Questions

Open a GitHub discussion or issue when you need help understanding expected behavior, queue choice, or integration patterns. Include enough context for the maintainer to reproduce or reason about the question.

## Bugs

Open a bug report with:

- `osiiso` version or commit.
- Python version.
- Operating system.
- Queue backend: `AsyncQueue`, `ThreadQueue`, or `ProcessQueue`.
- Minimal reproduction.
- Expected and actual behavior.

## Feature requests

Open a feature request with:

- The problem you are trying to solve.
- Why existing APIs are not enough.
- A short example of the desired usage.
- Any compatibility or migration concerns.

## Security

Do not open public issues for vulnerabilities. Follow [SECURITY.md](SECURITY.md).
