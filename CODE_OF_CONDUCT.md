# Code of Conduct

This project should be a practical, respectful place to discuss software.

## Expected behavior

- Be direct, factual, and respectful.
- Assume good intent, but accept correction when impact differs from intent.
- Keep technical disagreement focused on evidence, maintainability, user impact, and project goals.
- Give enough context for others to understand your issue, proposal, or review.
- Respect maintainer decisions about scope, priority, and release timing.

## Unacceptable behavior

- Harassment, threats, insults, or sustained personal attacks.
- Discriminatory language or behavior.
- Posting private information without permission.
- Repeatedly derailing issues, pull requests, or discussions.
- Bad-faith participation, spam, or intentionally disruptive behavior.

## Enforcement

Maintainers may remove comments, close issues or pull requests, block users, or take other reasonable action to keep the project healthy.

If you need to report a conduct issue, contact the maintainer at `ichingasamuel@gmail.com`. Include relevant links, screenshots, and a clear description of what happened. Reports will be handled with as much confidentiality as practical.
